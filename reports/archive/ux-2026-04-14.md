## Top 3 Visual Issues (unchanged from Apr 13, re-stacked by severity)

### 1. Type scale is chaos → nothing looks like it belongs
26 font sizes across 7,038 lines. Visual hierarchy is accidental. This is the most visible "amateur" signal on the app.

**Fix (paste at top of `app.jsx`, near `const F`):**
```js
const T = { xs:11, sm:12, base:13, md:14, lg:15, xl:18, h3:22, h2:28, h1:40 };
```
Then sweep: `fontSize:10`→`T.xs`, `fontSize:16`→`T.md`, `fontSize:20`→`T.xl`, `fontSize:24`→`T.h3`, etc. Kill 17 sizes in one pass.

### 2. Emoji in chrome — now with a 219-occurrence floor
Not 34. **219**. Hearts, planes, fires, pins, stars, sports icons, sparkles, waves, mountains, lightning, rockets, money bags, bells, phones, cameras — the whole Unicode arsenal. Every one of these is a tell that Peakly is a side-project and not a $79/yr product.

Jony Ive asks "what does this pictogram add that the label doesn't?" 95% of the time: nothing. Replace with lucide-react or custom 1px SVG strokes. CDN `lucide` is fine (no build step).

**Hard rule going forward:** emoji in `VENUES` tiebreaker strings is OK (user-facing poetry like "🌊 glassy lines"). Emoji in chrome (buttons, nav, tabs, badges, inputs) — banned.

### 3. Photo overlays crush dark photos (0.55–0.75)
Dusk and night venues become title-less black rectangles. Airbnb sits at 0.25–0.35. Fix (4 one-liners, ~2min):
```jsx
// app.jsx:1906 ListingCard
background:"linear-gradient(to top, rgba(0,0,0,0.38) 0%, transparent 52%)"
// app.jsx:2056 FeaturedCard
background:"linear-gradient(to top, rgba(0,0,0,0.40) 0%, transparent 55%)"
// app.jsx:2143 CompactCard
background:"linear-gradient(to top, rgba(0,0,0,0.40) 0%, transparent 50%)"
// app.jsx:3189 Hero
background:"linear-gradient(to top, rgba(0,0,0,0.35) 0%, transparent 60%)"
// app.jsx:5510 (0.75 → 0.45 max)
background:"linear-gradient(to top, rgba(0,0,0,0.45) 0%, transparent 50%)"
```

## Top 3 Interaction Issues

### I1. Book CTA is BLUE, violating the March 23 decision
Lines 2025, 2110. Decision doc says `#222`. Shipped code says blue gradient. Two lines to change:
```jsx
// replace on 2025 and 2110:
background:"#222", border:"1px solid #000",
```
Keep border-radius 20. Remove the gradient. Dark CTAs test better for subscription pricing pages (Linear, Vercel, Stripe all dark).

### I2. Touch targets below 44px — Book button 36, hearts 36, share 32
Book button is the most important UI element in the app. Scheduling `minHeight:44` along with all heart/share icons is a 5-minute sweep:
```bash
# Find all <44 targets:
grep -nE "minHeight:(3[0-9]|4[0-3])" app.jsx
```
Currently 3 violations surfaced; full sweep likely finds ~8–12.

### I3. Loading states look like errors
Shimmer on hero + `color:#aaa` condition text with no animation = dead UI, not loading UI. Users bounce thinking the app's broken. Add a 1.2s shimmer keyframe + swap `#aaa` for `#0284c7` pulsing at 0.6 opacity.

```css
/* add to CSS injection block ~line 80 */
@keyframes condPulse { 0%,100%{opacity:0.4} 50%{opacity:0.9} }
.condLoading { animation: condPulse 1.4s ease-in-out infinite; color:#0284c7; }
```

## Inspiration Steal

**Airbnb's ListingCard is the template.** Specific thing to steal: their photo-indicator dots when a card has multiple photos (tiny white dots at bottom-center, 4–6px, 0.35 inactive / 1.0 active). Peakly's cards are single-photo dead-ends. Adding 3–5 swipeable shots per venue + the dots turns the Explore feed into a browse-worthy experience instead of a list. This is the #1 feature to ship after design-system normalization.

## Decision Made

**Freeze net-new UI work until the design-token pass lands.** No new screens, no new sheets, no new badges shipped in `app.jsx` until:
- `T` + `R` + `C` tokens exist
- Type scale collapsed from 26 → 9 sizes
- All primary CTAs are `C.cta`/`#222`
- Top-of-card overlays are ≤ 0.40

This is a ~90 minute change. Every JSX added without it makes the cleanup 10% harder. Cutting additions now is cheaper than refactoring 7K lines in two weeks.
