# UX Audit — 2026-04-05

**UX Score: 6/10**

Good bones. The core flow works and the visual language is consistent. But there are real problems that cost conversions: undersized touch targets, a 5-tap onboarding wall, duplicate price display, and a BottomNav that fails basic accessibility. The single biggest miss is that a brand-new user has to tap through 3 onboarding screens before they can see a single venue with conditions.

---

## 1. CORE FLOW AUDIT -- Taps to First Venue with Conditions

**Current flow for a fresh visitor:**

1. Splash screen (auto, ~1.8s wait)
2. Onboarding step 0 -- "Get Started" (TAP 1)
3. Onboarding step 1 -- select airport, enter name/email, "Continue" (TAP 2, TAP 3 for airport selection)
4. Onboarding step 2 -- select sports, "Show me what's firing" (TAP 4, TAP 5)
5. Explore tab loads, weather fetches, hero card appears
6. Tap hero card or any venue card (TAP 6)
7. VenueDetailSheet opens with conditions + flight price visible

**Result: 6 taps to a venue with conditions visible. Airbnb benchmark: 3.**

But it is worse than it looks. The user must WAIT during step 5 while weather data fetches in batches. The hero card shows a shimmer skeleton until weather resolves. If weather is slow, the user sees "Checking conditions..." text on cards with no score. The perceived time-to-value is 6 taps + 3-10 seconds of loading.

**The fix is structural**: skip-to-explore should be the DEFAULT. Onboarding should be deferred or inline. See "The One Thing" below.

## 4. THE ONE THING

**The single highest-impact UX change: Let first-time users see venues IMMEDIATELY. Defer onboarding to after first engagement.**

Right now, a new user hits a 3-screen onboarding wall before seeing any content. The onboarding auto-shows after 900ms (line 10497). The user sees: splash -> onboarding modal -> 3 steps -> finally the explore tab. This is a 6-tap, 15+ second journey to see a single venue.

Airbnb shows you listings instantly. Profile setup happens when you try to book.

**The fix:** Remove the auto-show onboarding. Instead, show the explore tab immediately with a default airport (use geolocation which is already implemented). Show a subtle inline prompt in the header area only after the user has tapped on 2+ venues, asking them to set their home airport for accurate prices.

Here is the complete code change:

```jsx
// --- FILE: app.jsx ---
// --- CHANGE 1: Remove auto-show onboarding (line ~10494-10499) ---
// REPLACE:
  // Auto-show onboarding for first-time visitors (slight delay so Explore tab renders first)
  useEffect(() => {
    if (!profile.hasAccount) {
      const t = setTimeout(() => setShowOnboarding(true), 900);
      return () => clearTimeout(t);
    }
  }, []); // eslint-disable-line react-hooks/exhaustive-deps

// WITH:
  // Track venue opens for deferred onboarding prompt
  const venueOpenCount = useRef(0);
  const [showInlineSetup, setShowInlineSetup] = useState(false);

  // Deferred onboarding: after user opens 2 venues, show inline airport prompt
  const openDetailWithTracking = useCallback(listing => {
    openDetail(listing);
    if (!profile.hasAccount) {
      venueOpenCount.current += 1;
      if (venueOpenCount.current >= 2 && !showInlineSetup) {
        setShowInlineSetup(true);
      }
    }
  }, [openDetail, profile.hasAccount, showInlineSetup]);
```

```jsx

// After the SearchBar div in the Explore header (after line ~10859), add:

              {showInlineSetup && !profile.hasAccount && (
                <div className="bounce-in" style={{
                  margin:"8px 0 0", padding:"10px 14px",

                  borderRadius:14, display:"flex", alignItems:"center", gap:10,

                  <span style={{ fontSize:18 }}>plane emoji</span>
                  <div style={{ flex:1 }}>

```

```jsx

```

**Impact:** New users see venues with conditions in 0 taps (auto on load). First real interaction is tapping a venue card (1 tap). After 2 venue opens, a subtle prompt asks them to personalize. This matches the Airbnb pattern: show value first, capture info second.

**Tap count after fix: 1 tap to first venue with conditions visible.**
