### New Issues Found This Cycle

**Issue N1: No hero image loading skeleton on VenueDetailSheet**
Line 7282: Hero `<img>` starts at `opacity:0` and fades in on load. But there's no background gradient shown while loading (unlike ListingCard at line 3991 which falls through to `listing.gradient`). Users see a black rectangle for 1-3 seconds on slow connections. Previous report Fix #10 addressed this — still not shipped.

**Issue N2: ListingCard "Book" button uses blue gradient, not #222**
Line 4089: `background:"linear-gradient(135deg,#1a56db,#0ea5e9)"`. The CLAUDE.md design decision (2026-03-23) says "CTA buttons use #222 (dark), not blue." The sticky CTA in VenueDetailSheet correctly uses `#222` (line 7625). The ListingCard "Book" button contradicts the design system.

**Issue N3: VenueDetailSheet hero loads with `loading="lazy"`**
Line 7282: `loading="lazy"` on the hero image. This is the primary visual element — it should use `loading="eager"` since it's above the fold. `lazy` causes a blank hero for a beat after the sheet animates in, especially on slower connections.

**Issue N4: Photo duplication at 2,226 scale**
176 unique Unsplash photo IDs across 2,226 venues. Each base image is reused ~12.6 times with different crop parameters. Users browsing through multiple venues in the same category will notice identical photos with slight crops. This was acceptable at 192 venues — it's jarring at 2,226.

**Issue N5: "Conditions now" label uses #aaa**
Line 7342: `color:"#aaa"` on the "CONDITIONS NOW" uppercase label above the score. Ratio against #f0f9ff background: ~2.1:1 (fails AA). Against #f7f7f7: ~2.3:1 (fails AA). This is a new contrast failure not in the original 11.

## 4. COMPETITOR COMPARISON

### Top 3 remaining visual gaps vs. Airbnb/KAYAK/Surfline

**Gap 1: Typography scatter (REPEAT)**
VenueDetailSheet uses font sizes: 9, 10, 11, 12, 13, 14, 16, 20, 22, 26px — 10 distinct sizes in one component. Airbnb uses ~5 sizes with clear hierarchy. Consolidate to: 10, 12, 14, 18, 22px.

**Gap 2: Section spacing rhythm is irregular (REPEAT)**
VenueDetailSheet section spacing: `marginBottom:14` for score cards, `marginBottom:16` for forecast/tips/gear/experiences, `marginBottom:14` for Booking.com/SafetyWing CTAs, `marginBottom:16` for Peakly Pro and Save to list. It's a mix of 14 and 16. Pick one — 16px — and apply uniformly.

**Gap 3: No empty states for similar venues**
If a venue has no similar venues (e.g., only venue in its category for a region), the "You'd also like" section simply doesn't render. This creates an abrupt layout gap between the rating/tags and the insider tips. Consider always showing the section with a fallback (e.g., "More [category] spots coming soon").

### Inspiration: Steal from AllTrails

AllTrails' trail detail page has a **"Quick Facts" horizontal chip row** immediately below the hero. Compact pills showing: difficulty, length, elevation gain, route type, estimated time. Peakly has all this data (tags, rating, reviews) but shows them in a full-width section. Compressing into a single horizontal chip row would free vertical space and make the conditions/flight cards visible without scrolling on smaller phones.

## 7. Decision Made

**Show all category pills, remove "+ More" gate.**

The collapsed pill bar was designed when Peakly had 192 venues and 3 "important" categories. Now there are 2,226 venues across 12 categories. The pill bar already supports horizontal scrolling — there's zero technical reason to hide 8 categories behind a tap. Users who came for diving, climbing, or kayaking literally can't find their sport without an extra tap. Sport-ordered tabs (shipped today) already puts the user's favorites first — let the rest be discoverable by scrolling right. This is a 3-line code change with zero layout risk.

Ship **Fix 2: the WCAG contrast batch**. 13 color value changes. Each one is a single string replacement. Combined time: under 10 minutes. Zero layout risk. Zero design review needed.
