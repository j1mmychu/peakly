Line 8509: `color: active === t.id ? "#0284c7" : "#b0b0b0"`. The inactive tab color `#b0b0b0` on `#fff` background is 2.13:1. Needs to be at least `#767676` for AA compliance. This is the bottom nav — the most-used UI element in the app. Every user sees this on every screen.

Lines 5328-5333: The saved venues inline carousel shows only `l.gradient` background — no photos rendered despite `l.photo` being available on all 2,226 venues. Every other card surface in the app shows photos. This one got missed.

## 4. COMPETITOR COMPARISON

### Top 3 remaining visual gaps vs. Airbnb/KAYAK/Surfline

**Gap 1: Typography scatter (REPEAT — 2nd report)**
VenueDetailSheet uses font sizes: 9, 10, 11, 12, 13, 14, 16, 20, 22px — 9 distinct sizes in one component. Airbnb uses ~5 sizes with clear hierarchy. Consolidate to a type scale: 10, 12, 14, 18, 22px.

**Gap 2: No loading shimmer on VenueDetailSheet hero image (REPEAT — 3rd report)**
Line 7557: Hero image starts at `opacity:0` with no gradient placeholder behind it. Users see a blank/black rectangle for 1-3s while the photo loads. ListingCard (line 4111) correctly falls back to `listing.gradient`. VenueDetailSheet should too.

**Gap 3: Sticky CTA bar lacks safe-area inset**
Line 7862-7896: The sticky CTA bar at bottom of VenueDetailSheet uses `padding:"12px 16px 18px"`. On iPhones with home indicators, the bottom 34px is the safe area. The CTA sits too close to the home indicator. Should use `paddingBottom: "max(18px, env(safe-area-inset-bottom, 18px))"`.

### Inspiration: Steal from Surfline

Surfline's spot detail page shows a **conditions confidence indicator** — a small badge reading "High Confidence" or "Moderate Confidence" next to the forecast. Peakly's scores are black boxes to users. Adding a one-word confidence label (based on how much weather data is available — e.g., marine data present vs absent) would build trust in the scoring system. This directly supports the "Window Score" vision from Phase 2.

## 7. Decision Made

**Standardize BottomNav inactive color to #767676.**

The bottom navigation bar is the single most-used UI element in any tab-based app. Every user sees it on every screen. The current inactive color (#b0b0b0) fails WCAG AA at 2.13:1 contrast against the white background. The fix is a single color value change. #767676 is the lightest gray that passes AA (4.54:1) and is the industry standard for "muted but readable" — used by iOS tab bars, Material Design, and Airbnb. This is a one-character fix that affects the most-viewed element in the entire app.
