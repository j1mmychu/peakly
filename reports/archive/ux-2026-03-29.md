# Peakly UX Audit — 2026-03-29

## Design Score: 6.8/10

Down 0.2 from yesterday's 7.0. Reddit soft launch is March 31 — two days out. Seven issues flagged across five consecutive reports remain unfixed. This is the last pre-launch audit. If these ship as-is, first-impression users will encounter undersized tap targets, inconsistent CTA colors, missing price context, and disorienting scroll behavior. All fixable in under 10 minutes of actual code changes.

---

## Top 3 Visual Issues

### 1. Emoji in UI chrome violates own design rules (REPEAT — 5th report)

**FILE:** app.jsx
**LINES:** 164–176 (CATEGORIES array), 4884 (SearchBar search icon), 4182 (flight icon), 4904 (alert banner)
**SEVERITY:** Medium-High (brand perception)

CLAUDE.md explicitly states: "Remove emoji from UI chrome. Emoji in category pills, section headers, and badges look amateur. Clean text + SVG only." Yet CATEGORIES still has emoji in every entry, SearchBar still renders `🔍`, flight badges still use `✈️`, and the alert banner still uses `📍`.

The app uses ~163 emoji instances across the codebase. For a "Steve Jobs good" app competing with Airbnb and Surfline — neither of which uses emoji in primary UI chrome — this reads as a prototype, not a product.

**FIX:** Phase 1 (pre-launch, surgical):
- Remove emoji from category pill rendering at line 5262: change `{c.emoji} {c.label}` to just `{c.label}`
- Replace `🔍` in SearchBar (line 4884) with an SVG search icon matching BottomNav style
- Replace `✈️` in flight badges with a small SVG plane icon

### 2. CTA button color still inconsistent across 3 card types (REPEAT — 4th report)

**FILE:** app.jsx
**LINES:** 4249 (ListingCard), 4321 (FeaturedCard), 5455 (hero CTA)

CLAUDE.md says CTA buttons use `#222`. Reality:
- ListingCard Book: `linear-gradient(135deg, #1a56db, #0ea5e9)` — blue gradient
- FeaturedCard Book: same blue gradient
- Hero CTA View Details: `#0284c7` — flat sky blue

Three styles. Should be one. This is a 30-second find-and-replace.

**FIX:**
```jsx
// Lines 4249, 4321: replace gradient
background: "#222",

// Line 5455: replace flat blue
background: "#222",
```

### 3. No round-trip/one-way clarification on any flight price (NEW)

**FILE:** app.jsx
**LINES:** 4183, 4316, 5446, 7668+ (every flight price display)

Every card shows `$XXX` with no indication of whether it's one-way or round-trip. Travelpayouts returns one-way prices. Users seeing "$189" who assume round-trip will feel deceived on click-through. This is a trust-killer on day one with Reddit users who will scrutinize everything.

**FIX (minimal):** Add a subtle label beneath or beside every price:
```jsx
// After price span in ListingCard (line 4184):
<span style={{ fontSize: 8, color: "#999", fontWeight: 500 }}> r/t</span>
// Or if one-way:
<span style={{ fontSize: 8, color: "#999", fontWeight: 500 }}> each way</span>
```

This needs a decision from Jack: are Travelpayouts prices one-way or round-trip? Verify, then label accordingly everywhere.

## Top 3 Interaction Issues

### 1. No scroll-to-top on category switch (REPEAT — 4th report)

**FILE:** app.jsx
**LINE:** 5250
**COMPONENT:** ExploreTab pill onClick

Users deep in Surf scroll position switch to Ski/Board and land mid-list. One line of code. Flagged since March 26.

**FIX:**
```jsx
scrollRef.current?.scrollTo({ top: 0, behavior: 'instant' });
```

### 2. Category pill touch targets 36% below minimum (REPEAT — 5th report)

**FILE:** app.jsx
**LINE:** 5253
**COMPONENT:** ExploreTab category pills

`padding: 5px 6px` with `fontSize: 11` = ~28px tall. Apple HIG minimum: 44px. This will get called out instantly in any Reddit feedback thread. Adventure travelers use these on lifts, boats, and trails — gloved/wet fingers need generous targets.

**FIX:**
```jsx
padding: "8px 10px",
fontSize: 12,
```

### 3. BottomNav tab labels at 10px font size (REPEAT — 2nd report)

**FILE:** app.jsx
**LINE:** 8565

`fontSize: 10` for nav labels is below readable minimum for many users. iOS tab bars use 10pt (which renders larger than CSS 10px due to point vs pixel). Combined with `padding: 8px 0`, effective touch area is ~40px — below 44px minimum.

**FIX:**
```jsx
// Line 8555
padding: "10px 20px",
minWidth: 64,
minHeight: 44,

// Line 8565
fontSize: 11,
```

## Pre-Launch UX Risk Assessment (March 31 Reddit)

Reddit users are notoriously detail-oriented. Here's what they'll call out first:

1. **"Prices don't say if they're round-trip"** — instant trust loss
2. **"Why do the buttons look different on every card?"** — inconsistency reads as unfinished
3. **"The emojis make it look like a school project"** — brutal but accurate for the target demo (serious adventure travelers aged 25-40)
4. **"I can barely tap these category buttons"** — especially on smaller Android devices
5. **"The scroll position doesn't reset when I switch categories"** — confusing navigation

Items 2, 4, and 5 are literally seconds to fix. Item 3 is 5 minutes. Item 1 needs a data check + 30 seconds of code.

## What's Actually Good

Credit where due — the foundation is strong:

- **Photo integration is excellent.** Lazy-loaded Unsplash photos with shimmer states and onError fallbacks. This looks professional.
- **Sheet animations feel native.** The spring curves (0.32, 1.2, 0.4, 1) on sheet-up and the swipe-to-dismiss on VenueDetailSheet are buttery.
- **Hero card is well-designed.** "Your Best Window Right Now" with condition score + flight price + CTA is a strong hook.
- **Skeleton loaders work.** Shimmer animations during data load prevent jarring content shifts.
- **Pull-to-refresh.** Native-feeling gesture with spinner. Good mobile pattern.
- **Heart save animation.** The `heart-pop` keyframe with overshoot is satisfying.

---

## Inspiration

**Steal from Surfline: the condition confidence indicator.** Surfline shows not just the forecast, but a confidence bar — "High confidence" / "Moderate confidence" — next to each day's rating. Peakly's scores feel authoritative but users have no way to gauge reliability. Adding a subtle "High / Med / Low confidence" label based on forecast horizon distance (today = high, day 5 = medium, day 7 = low) would massively increase trust. No API changes needed — just compute from the day index.

---

## Decision Made

**All flight prices must include trip-type label before Reddit launch.** Whether round-trip or one-way, every price display ($XXX) needs a small clarifying label. This is non-negotiable for trust. Users who click through and see a different number than expected will bounce and never return. One small label prevents that. Jack needs to verify Travelpayouts price format (one-way vs round-trip) and the label gets added to all 4 card types + detail sheet in under 2 minutes.

## Summary

Two days to Reddit. The app's bones are genuinely good — photos, animations, data architecture, scoring. But seven polish issues have been flagged across five reports and none deployed. The total fix time for all seven is under 10 minutes. The risk isn't that Peakly is bad — it's that small rough edges compound into a "this isn't ready" impression for detail-oriented Reddit users.

**If I could ship three fixes before Monday:**
1. Scroll-to-top on category switch (1 line)
2. Category pill padding increase (2 numbers)
3. CTA button color standardization to `#222` (find-replace)

These three take 60 seconds combined and eliminate the most visible inconsistencies.

**Design Score: 6.8/10** — down because proximity to launch makes unfixed issues more costly, not because the app degraded. The ceiling is 8.5+ if the persistent backlog gets cleared. The floor is wherever Reddit puts it on Monday.
