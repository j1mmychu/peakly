## Explicit Product Decisions This Report

**DECISION 1: Wishlists + Trips tabs — DEFER to 1,000 users.**  
CLAUDE.md flagged "ACTION NEEDED: Expose Trips and Wishlists tabs." This is wrong. 3-tab nav (Explore → Alerts → Profile) is correct at zero users. The core conversion is Explore → Detail → Book. Every extra tab is distraction for new users who don't understand the app yet. Revisit at 1K MAU.

**DECISION 2: Capacitor iOS App Store path — DEFER 60 days.**  
Infrastructure is ready. But Apple Developer enrollment ($99) + Xcode build time + App Store review (1-4 weeks) = 45-60 days minimum. Reddit validates whether we have product-market fit first. If Reddit generates 500 users, iOS is justified. If Reddit generates 50 users, iOS is not the problem. **Do not spend $99 or developer time on this until post-Reddit data is in.**

**DECISION 3: Strike alerts server polling — DEFER 30 days.**  
Push infra is built. But you cannot send strike alerts to zero subscribers. Build the subscriber base first (email capture, Reddit). After 100 active users with alerts set, implement the VPS polling endpoint. Building server infrastructure for alerts before the alert list has users is waste.

**DECISION 4: Open-Meteo rate limit risk — MONITOR (not SHIP).**  
DevOps has flagged this repeatedly. At Reddit launch with 200-500 new users in 48 hours, batched weather fetching + 30-min localStorage cache should hold. If we see 429s in Sentry, respond same day. Do NOT buy Open-Meteo commercial plan ($29/mo) until we actually hit limits. This is premature optimization.

**DECISION 5: Venue expansion — FREEZE at 3,726.**  
Adding more venues does not make us better. We now have more venues than we have unique photos by a factor of 21x. Quality ceiling is hit. No venue expansion until photo diversity is solved. FREEZE.

## Success Criteria & 90-Day Projection

**What defines success?**
- 100 users in first 48 hours post-Reddit = minimum viable signal
- 30-day retention >20% = product is habit-forming
- >5% of users set an alert = core value prop resonating
- At least 1 flight click per session = monetization funnel working

**What has to be true for 8K users at 90 days (not 5K)?**

The 5K→8K gap requires at least one of:
1. **Influencer hit** — One TikTok or Instagram post from a surf/ski micro-influencer (50K+ followers) showing a real window that paid off. This is not fully controllable but can be seeded by identifying 10 influencers pre-launch.
2. **Organic retention loop** — 30-day retention above 30% means users return before each weekend trip. At 30% retention, word-of-mouth compounds. Below 20%, the app is a novelty.
3. **SEO traffic kicking in** — At 95% SEO score and sitemap live, Google indexing should generate 50-100 organic visits/day by day 60. This alone is 1,500-3,000 additional users over 90 days.
4. **Strike Mission moment** — One genuine "you had to be there" event (surprise powder dump, unexpected swell) that Peakly called before mainstream surf/ski apps. If we tweet/post this with receipts, it's the single best growth moment possible.

The difference between 5K and 8K is **retention + one viral moment.** Both require the product to be actually good, not just launched.

## The Product Risk Nobody Is Talking About

**The scoring algorithm is now frozen — but it was never validated.**

Every product decision, every "Epic" badge, every "Best Right Now" ranking is driven by `scoreVenue()`. We froze the algorithm "for stability before Reddit." But we never ran a user test asking: "Does this score match what you'd actually experience?" 

If surfers land on the app and see a beach break with a "92 — Firing" score during onshore mush, they will never return. Scoring accuracy is not a Phase 2 problem — it is the product. One wrong score shown to a core user (someone who actually surfs or skis and therefore is our target) poisons the well permanently.

**Recommended action:** Before Reddit, manually check the top 3 surf venues and top 3 ski venues in today's "Best Right Now" against actual current conditions on Surfline/OnTheSnow. If scores are directionally correct, proceed. If they're embarrassingly wrong, this is a P0 that no amount of splash screen polish fixes.
