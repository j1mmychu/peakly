## One Product Risk Nobody Is Talking About

**Peakly has no feedback loop. There is no way to know what users think.**

There is no in-app feedback mechanism. No "was this helpful?" prompt. No NPS survey. No way for a user to report a wrong tag, a bad score, or a broken flight link. The only feedback channel is Reddit comments -- which are public, permanent, and uncontrolled.

This means:
- If venue tags are wrong, Peakly learns about it from a public roasting, not a private report
- If scoring feels off, there is no signal -- users just leave
- If flight links are broken (TP_MARKER, VPS down), users click, get a bad Aviasales page, and Peakly never knows
- The team is flying blind on what works and what doesn't

Every conditions app that scaled past 10K users (Surfline, OpenSnow, AllTrails) built feedback loops early. Surfline has "Report Conditions." AllTrails has trail reviews. OpenSnow has community snow reports. These aren't features -- they are data infrastructure for product improvement.

**Recommendation:** Add a minimal "Report an issue" link in VenueDetailSheet footer. It opens a pre-filled `mailto:` with venue ID, score, and category. Zero backend. 10-minute implementation. This turns every expert user from a potential public critic into a private contributor. Ship it before Reddit.
