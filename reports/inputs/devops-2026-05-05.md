### 3. Supabase SQL deploy — share-a-list

The commit message ends with "REQUIRES SQL DEPLOY before share button works." There is no `*.sql` file in the repo. The client calls:

- `client.from("shared_lists").insert({slug, owner_id, source_list_id, name, emoji, venue_ids})` (app.jsx:1861)
- `client.from("shared_lists").select(...).eq("slug", ...).maybeSingle()` (anon-readable, app.jsx:1898)
- `client.rpc("bump_share_view", { p_slug: slug })` (app.jsx:1908 — fire-and-forget view counter)

Until the table + RPC exist in Supabase, every share button click `INSERT`s and gets a permission/relation error. Recipients hitting `?l=<slug>&r=<owner>` URLs see "list not found" silently.

**Action item:** add a canonical migration file at `server/sql/share-lists.sql` so the deploy is reproducible. Suggested shape (Jack to verify against the actual table he creates in the Supabase dashboard):

```sql
-- server/sql/share-lists.sql
create table if not exists public.shared_lists (
  slug            text primary key,
  owner_id        uuid not null references auth.users(id) on delete cascade,
  source_list_id  text not null,
  name            text not null,
  emoji           text default '🗺️',
  venue_ids       text[] not null default '{}',
  view_count      int  not null default 0,
  created_at      timestamptz not null default now()
);

alter table public.shared_lists enable row level security;

-- Anyone (anon) can read a shared list by slug.
create policy "shared_lists_anon_read" on public.shared_lists
  for select using (true);

-- Only the owner can insert their own list.
create policy "shared_lists_owner_insert" on public.shared_lists
  for insert with check (owner_id = auth.uid());

-- Only the owner can delete (no UPDATE for now — snapshots are immutable).
create policy "shared_lists_owner_delete" on public.shared_lists
  for delete using (owner_id = auth.uid());

-- View-count bump callable by anon, but only via this SECURITY DEFINER RPC
-- so anon can't write to view_count directly.
create or replace function public.bump_share_view(p_slug text)
  returns void
  language sql
  security definer
  set search_path = public
as $$
  update public.shared_lists set view_count = view_count + 1 where slug = p_slug;
$$;
grant execute on function public.bump_share_view(text) to anon, authenticated;
```

Time to land in Supabase SQL editor: ~3 min. Time to commit the migration: ~30 sec.
