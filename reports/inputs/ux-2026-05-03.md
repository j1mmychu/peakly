## Top 3 Visual Issues (May 3)

### 1. The dark-CTA rule still has one believer (1/6 surfaces honor it)

CLAUDE.md, 2026-03-23: *"CTA buttons: `#222` (dark, NOT blue)."*

Inventory of primary CTAs as of HEAD:

| Surface | Color | Line |
|---|---|---|
| ListingCard "Book" | `linear-gradient(135deg,#1a56db,#0ea5e9)` | 2124 |
| FeaturedCard "Book" | `linear-gradient(135deg,#1a56db,#0ea5e9)` | 2209 |
| Hero "View Details" | `#0284c7` | 3342 |
| Alerts empty "Create your first alert" | `#0284c7` | 4037 |
| Onboarding final "Show me what's firing" | `linear-gradient(...)` + glow | 5404 |
| VenueDetailSheet sticky Flights CTA | `#222` ✅ | 5909 |

**The onboarding handoff is the worst.** New user taps "Show me what's firing" — blue gradient with `boxShadow:"0 4px 20px rgba(2,132,199,0.4)"` glow. Three seconds later they tap a venue, see the dark sticky CTA, and the design language has already broken. First impression of "this app's button system" is "inconsistent."

### 2. Three flight-price colors across three card surfaces

Same number, three colors:

- Best-Right-Now carousel (L3411): `#0284c7` (sky blue)
- ListingCard / FeaturedCard price: `#222` (dark)
- VenueDetailSheet flight panel (L5685): `#16a34a` (green)

Tap from carousel → detail and the price hue mutates mid-flow. This isn't semantic color — it's three writers without a style guide.

### 3. NEW — `fontSize:8` exists in production

Line 2279, FeaturedCard breakType label: `fontSize:8`. There is exactly one 8px text element in the app and it's a content label, not a decorative caption. Apple HIG floor is 11; W3C WCAG considers anything below 12 borderline. Combined with the **17 instances of `fontSize:9`** (carousel "rt", trending tag, slider end-stops, forecast labels), there's a real legibility problem on the smaller cards.

The type scale audit:

```
9px:  17 instances    11px: 78 instances    14px: 55 instances    20px: 12 instances
10px: 52 instances    12px: 81 instances    15px: 21 instances    22px:  8 instances
                      13px: 90 instances    16px: 16 instances    24px+: rest
```

A clean six-step scale would be **11 / 13 / 15 / 18 / 22 / 28**. Current is 16 sizes for body text. The 9px usages are the urgent fix — they're not just inconsistent, they're unreadable in sun.

## Top 3 Interaction Issues

### I1. Share button still 32px, still `↑`

L2010: `width:32, height:32`, child glyph `↑` (U+2191, "up arrow"). This has been flagged on Apr 12, 13, 14, 15, 16. Apple HIG minimum tappable area is 44×44pt. 32px on a phone is missable, and `↑` is not a recognized share glyph on iOS or Android. A 4-line fix.

### I2. Best-Right-Now carousel images pop in hard

L3397: `<img>` has no opacity start state, no `onLoad` handler. Every other card surface uses `opacity:0 → onLoad → opacity:1` over `0.35s`. On 3G this carousel pops the photo in over the gradient with no transition; on Wi-Fi it's a flicker. Pattern is already in the codebase elsewhere — this is a copy-paste of two attributes.

### I3. Airport change has no acknowledgment

Profile → change home airport. Prices recompute reactively (correct), but with no confirmation the user can't tell if the prices they're seeing are stale or new. On a 3-second flight-fetch, the only signal is the `from $X` numbers blinking — easy to miss. A 1.5-second toast ("Prices updated from LAX") closes the loop. Two state hooks, no design change required.

## Inspiration Steal

**Airbnb's "Where to next?"** — the front-page horizontal scroller of curated regions has a single border-radius (16), a single overlay opacity (0.25), and a single text size for the city label (16). Every card looks like a sibling of every other card. Peakly's Best-Right-Now carousel has the bones of this pattern but currently uses a different photo treatment (no fade), a different price color, and a different border-radius from the cards below it. **Steal Airbnb's discipline, not their content.** Make all card surfaces obey one set of rules.

## Decision Made

**Define a `THEME` object and reference it instead of inline literals — but don't ship it yet.**

Sketch:
```js
const THEME = {
  cta: "#222",                                  // primary action
  brand: "#0284c7",                             // brand only — never on a button
  go: "#22c55e", maybe: "#eab308", wait: "#ef4444",
  textBody: "#222", textMuted: "#717171", textFaint: "#aaa",
  overlay: "rgba(0,0,0,0.32)",
  radius: { sm: 10, md: 14, lg: 20 },
  size:   { xs: 11, sm: 13, md: 15, lg: 18, xl: 22, hero: 28 },
};
```

A `THEME` const at the top of `app.jsx` would have prevented the four-CTA, three-price, twelve-fontSize chaos. **Don't ship the refactor pre-launch** — touching 90+ call sites violates "Do NOT over-engineer. Every fix is surgical." But add the const, document the values, and use it for any new component going forward. Pre-launch fixes stay surgical (Fixes 1–8 above).

Post-launch refactor: walk every `fontSize:` and `color:` literal in JSX, replace with `THEME.*`. That's a `Day After 1K Users` task.
