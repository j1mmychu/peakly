### 2. DEV: 4 surgical fixes, 30 minutes total, ship today

Each of these was identified 13+ days ago. None requires design input. None risks regressions. All are pure risk reduction:

**a) venue.facing (10 min):** `app.jsx:4683-4691` — delete the spotFacing/swellAlignment/swellAngleDiff block. Every surf break without an explicit `facing` property (the majority) defaults to 270° (west-facing). East-coast US breaks, Mediterranean spots, and Indo breaks get wrong swell alignment scores. Replace with: no directional multiplier until facing data exists in VENUES.

**b) fetchWeather 429 handler (5 min):** `app.jsx:4541` — current code: `if (!r.ok) throw new Error("weather fetch failed")`. Change to: `if (r.status === 429) return null; if (!r.ok) throw new Error(...)`. fetchMarine() at line 4558 already does this correctly. One line.

**c) React version pin (2 min):** `index.html:80-81` — change `react@18` → `react@18.3.1`, `react-dom@18` → `react-dom@18.3.1`. Unpkg resolves `@18` to "latest 18.x" — meaning a breaking minor release kills all users' apps simultaneously with no warning.

**d) Email capture to Loops.so (30 min):** In the onboarding completion handler, add a `fetch('https://app.loops.so/api/v1/contacts/upsert', ...)` POST with the user's email. Loops.so free tier handles 2,500 contacts. No LLC, no Stripe, no backend required. Every email entered since launch is gone unless this ships.

**Decision: SHIP all 4 as a single commit today. Bundle with TP_MARKER fix.**

### 3. DEV: Extract VENUES to venues.json — week's major engineering task

app.jsx is 2.0 MB. Approximately 1.4 MB is the VENUES array (~3,726 entries). Babel Standalone parses the entire file synchronously before React renders anything. On a mid-range Android at mobile data speeds — which is the profile of most Reddit/social traffic — this produces 5-10 seconds of blank white screen before any UI appears.

The fix is not a refactor. It is data extraction:
1. Copy the VENUES array out of app.jsx into `venues.json`
2. In app.jsx, initialize `venues` state as `[]` with a `venuesLoading` flag
3. In the root component's useEffect, `fetch('./venues.json').then(r=>r.json()).then(data => { setVenues(data); setVenuesLoading(false); })`
4. Show shimmer/skeleton on Explore tab while venuesLoading
5. app.jsx drops from 2.0 MB → ~0.6 MB. First Babel parse completes in ~1-2 seconds instead of 5-10.

This does not split the React app. app.jsx remains single-file. This is a data file extraction, not an architecture change. The venues.json fetch is a standard browser fetch, no bundler required.

**Decision: SHIP this week. This is the highest-ROI engineering task available. Estimated impact: 30-50% reduction in mobile bounce rate.**
